Why we need Fenwick Tree

The binary representation of a number

Note that Fenwick Tree starts from index 1, so we need to create a ‘tree’ with length n+1.

To create a Fenwick Tree, we need to call update() for each element in the original array, so the time complexity of creating a new Fenwick Tree is O(NlogN), which requires more time than creating a prefix array.

Fenwick Tree is usually used to calculate the arithmetic result (product, sum, but not min and max) of several elements in a certain range.

Leetcode 315 Count of Smaller Numbers After Self

You are given an integer array nums and you have to return a new counts array. The counts array has the property where counts[i] is the number of smaller elements to the right of nums[i].

Example:

Solution

The brute force solution is to do a linear scan of the array and for each element, we scan the elements behind it and calculate the count of smaller ones. This approach takes O(N^2) time complexity.

To make use of FenwickTree, we need to build a sorted map and reverse the array. Then, instead of counting the smaller numbers, we try to calculate the prefix sum of a element.
